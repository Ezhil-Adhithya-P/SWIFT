# SWIFT Ecosystem: Local Setup Guide

This guide provides instructions for setting up the SWIFT Smart Kiosk & Digital Wallet ecosystem on a local machine.

## 📋 Prerequisites
- **Node.js:** v18 or higher
- **MySQL:** v8.0 or higher
- **Expo Go:** Installed on your physical mobile device (for wallet testing)

---

## 🛠️ Step 1: Database Configuration
1. Ensure your MySQL server is running.
2. The system is configured to connect with the following credentials (defined in `server.js`):
   - **Host:** `127.0.0.1`
   - **User:** `root`
   - **Password:** `pass123`
3. The server will automatically create the `swift_db` database and all required tables upon first launch.

---

## 🚀 Step 2: Service Launch Sequence

Run each of these commands in a separate terminal:

### 1. Backend API Server
```bash
# In the root directory
node server.js
```

### 2. REC Kiosk (User Interface)
```bash
cd swift-kiosk
npm install
npm run dev
```

### 3. Vendor Management Console
```bash
cd swift-vendor
npm install
npm run dev
```

### 4. College Admin Dashboard
```bash
cd swift-admin
npm install
npm run dev
```

### 5. SWIFT Mobile Wallet (Expo)
```bash
cd swift-wallet
npm install
npx expo start --lan --clear
```

---

## 📱 Step 3: Mobile Connectivity (Crucial)
To test the Mobile Wallet on a physical phone:
1. Ensure your computer and phone are on the **same Wi-Fi network**.
2. Find your computer's LAN IP address (run `ipconfig` on Windows).
3. Update the `API_URL` in `swift-wallet/src/context/WalletContext.js` to match your IP:
   `const API_URL = 'http://YOUR_IP:3000/api';`
4. Scan the QR code generated by the Expo terminal.
